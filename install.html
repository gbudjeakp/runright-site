Overview

RunRight is a single binary that runs as a sidecar inside your existing CI job. It polls CPU and memory every 5 seconds, builds a p95 summary, and matches your usage against a catalog of 160+ AWS and 60+ GCP instance types to recommend the cheapest machine that still fits.

No SDK to import. No annotations. No config file required.

60-second quick-start (GitHub Actions)

- uses: gbudjeakp/run-right@v1
  with:
    job-id: ${{ github.job }}
    export: http
    http-url: https://your-runright-server

GitHub Actions

Add the action as the first step in any job you want to profile. It runs in the background and posts results when your job finishes.

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: gbudjeakp/run-right@v1
        with:
          job-id: ${{ github.job }}
          export: http
          http-url: ${{ secrets.RUNRIGHT_URL }}
          api-key: ${{ secrets.RUNRIGHT_API_KEY }}

      - uses: actions/checkout@v4
      # ... rest of your steps

PR comment

Set pr-comment: true to get an automatic recommendation comment on every pull request.

- uses: gbudjeakp/run-right@v1
  with:
    job-id: ${{ github.job }}
    export: http
    http-url: ${{ secrets.RUNRIGHT_URL }}
    pr-comment: true

Dry-run (gate on wrong sizing)

Use --dry-run to fail the job if the machine isn't right-sized — useful in pre-submit enforcement.

- name: Check machine sizing
  run: |
    runright monitor --dry-run --duration 0 \
      --export file --output-dir /tmp/rr

GitLab CI

runright-monitor:
  stage: .pre
  image: ubuntu:22.04
  script:
    - curl -fsSL https://github.com/gbudjeakp/run-right/releases/latest/download/runright-linux-amd64 -o runright
    - chmod +x runright
    - ./runright monitor --export http --http-url "$RUNRIGHT_URL" --job-id "$CI_JOB_NAME" &
    - echo $! > /tmp/rr.pid

after_script:
  - kill $(cat /tmp/rr.pid) 2>/dev/null || true

Jenkins

pipeline {
  agent any
  stages {
    stage('Monitor') {
      steps {
        sh '''
          runright monitor \
            --export http \
            --http-url "${RUNRIGHT_URL}" \
            --job-id "${JOB_NAME}" &
          echo $! > /tmp/rr.pid
        '''
      }
    }
    stage('Build') {
      steps { sh 'make build' }
    }
  }
  post {
    always { sh 'kill $(cat /tmp/rr.pid) 2>/dev/null || true' }
  }
}

Docker Compose

Run the full stack — backend, Postgres, and dashboard — with a single command.

git clone https://github.com/gbudjeakp/run-right
cd run-right
RUNRIGHT_API_KEY=your-key docker compose up -d

The dashboard is available at http://localhost:3000. The API is at http://localhost:8080.

Environment variables

Variable Description
RUNRIGHT_API_KEYBearer token for the API (optional in dev)
DATABASE_URLPostgres DSN (defaults to bundled postgres)
RUNRIGHT_SLACK_WEBHOOKSlack incoming webhook for notifications

Kubernetes

Deploy the backend as a Deployment with a ClusterIP service and a Secret for credentials.

kubectl create secret generic runright \
  --from-literal=api-key=your-key \
  --from-literal=database-url=postgres://...

kubectl apply -f https://raw.githubusercontent.com/gbudjeakp/run-right/main/k8s/runright.yaml

Helm (EKS / any K8s)

helm install runright ./helm/runright \
  --namespace runright \
  --create-namespace \
  --set config.apiKey=your-key \
  --set externalDSN=postgres://user:pass@host/db

With bundled PostgreSQL

helm install runright ./helm/runright \
  --namespace runright \
  --create-namespace \
  --set config.apiKey=your-key \
  --set postgresql.enabled=true \
  --set postgresql.auth.password=your-pg-pass

Terraform — ECS Fargate

module "runright" {
  source            = "github.com/gbudjeakp/run-right//terraform"
  vpc_id            = var.vpc_id
  private_subnet_ids = var.private_subnet_ids
  api_key           = var.runright_api_key
  db_password       = var.db_password
}

Creates: ECS cluster, Fargate task, RDS PostgreSQL, Secrets Manager entries, IAM roles, and security groups.

Terraform — EKS

module "runright_eks" {
  source       = "github.com/gbudjeakp/run-right//terraform/eks"
  cluster_name = var.eks_cluster_name
  api_key      = var.runright_api_key
  db_password  = var.db_password
  vpc_id       = var.vpc_id
  private_subnet_ids = var.private_subnet_ids
}

Uses the bundled Helm chart. Optionally creates an RDS instance and wires the DSN into a Kubernetes Secret.

CLI Reference

monitor

runright monitor [flags]

  --duration        Stop after this duration (0 = run until SIGTERM)
  --interval        Sampling interval (default 5s)
  --export          Backends: file, otlp, prometheus, http, slack
  --job-id          Job identifier
  --output-dir      Directory for file output (default .)
  --http-url        RunRight backend URL
  --slack-webhook   Slack incoming webhook URL
  --dry-run         Exit non-zero if machine is not right-sized

recommend

runright recommend [flags]

  --metrics         Path to metrics-summary.json (default metrics-summary.json)
  --provider        Filter: aws or gcp (default: both)
  --format          Output format: table, json, markdown

serve

runright serve [flags]

  --port            Listen port (default 8080)

  Env vars:
  DATABASE_URL          Postgres DSN
  RUNRIGHT_API_KEY      Bearer token for auth
  RUNRIGHT_SLACK_WEBHOOK  Webhook for weekly digest
